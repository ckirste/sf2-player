🖥 Server starten (einmalig einrichten)

Voraussetzung: Node.js installiert (LTS-Version reicht) 
# Einmalig — Abhängigkeiten installieren:
npm install

# Server starten:
node jam_server.js

# Ausgabe:
🎸 SF2 Jam Server läuft auf Port 8765

Tunnel für Internet-Mitspieler (localhost.run)

In einem zweiten Terminal — kein Account, kein Token nötig:

ssh -R 80:localhost:8765 nokey@localhost.run

# Ausgabe — diese URL an alle Mitspieler schicken:
your url is: https://abc123def.lhr.life
⚠️ Die URL ändert sich bei jedem Neustart des Tunnels — einfach per WhatsApp/Signal teilen.

🔌 Verbinden (alle Mitspieler)

sf2_player.html im Browser öffnen
Chrome oder Edge empfohlen (Firefox geht auch). Auf dem Handy: direkt im Browser öffnen.
SF2-Datei laden
Auf ＋ SF2 laden klicken. Alle Mitspieler sollten denselben Soundfont haben damit die Instrumente gleich klingen (z.B. FluidR3_GM.sf2 — kostenlos).
Jam-Panel öffnen — Sprungmarke 🌐 Jam in der Nav-Leiste
  • Name eingeben — wird allen angezeigt
  • Server-URL eintragen:
    • Gleiches WLAN: ws://192.168.X.X:8765
    • Über Internet: wss://abc123def.lhr.life
  • 🔌 Verbinden klicken

🎛 Mitspieler-Mixer

Nach dem Verbinden erscheint für jeden Mitspieler eine Zeile im Mixer:

ElementFunktion
Farbiger Punkt + Name Jeder Spieler hat eine eindeutige Farbe. Zeile blinkt kurz bei jeder Note.
Preset-Chips Zeigt welches Instrument der Mitspieler gerade hat.
■ Fett = exakt gleicher Sound geladen
≈ Grau = ähnlicher Sound (anderer SF2)
🎙 Indikator Leuchtet auf wenn der Spieler gerade spricht/singt (per Mikrofon)
🔊 Schieberegler (klein) Mikrofon-Lautstärke dieses Spielers
Großer Schieberegler MIDI-Noten-Lautstärke dieses Spielers
🔇 Button Diesen Spieler komplett stumm schalten (Noten + Mikrofon)

🎙 Mikrofon / Audio

Im Jam-Panel über dem Mixer:

EinstellungWann verwenden
🎤 Gesang / Headset Stimme, Headset, Freisprechen.
Echo-Unterdrückung ✅ · Rauschfilter ✅ · Auto-Gain ✅
🎸 Instrument / Line-In E-Gitarre (Kabel), Keyboard-Line-Out, Mikrofon vor Verstärker.
Echo-Unterdrückung ❌ · Rauschfilter ❌ · Auto-Gain ❌
(wichtig: Rauschfilter würde Gitarren-Sustain abschneiden)
Modus wählen (Gesang oder Instrument)
🎙 klicken → Browser fragt nach Mikrofon-Erlaubnis → erlauben
VU-Meter zeigt deinen Pegel. 🔇 zum vorübergehenden Stummschalten.
Latenz-Hinweis: Peer-to-Peer Audio über Internet = ca. 50–150ms. Für Gesang/Absprechen gut geeignet. Exaktes rhythmisches Zusammenspielen ist bei großen Entfernungen physikalisch begrenzt.

♩ Metronom-Sync

Das Metronom ist automatisch synchronisiert:

  • Wer BPM ändert → alle sehen und hören die neue BPM
  • Wer Start/Stop drückt → startet/stoppt bei allen
  • Neu verbundene Spieler bekommen den aktuellen Metronom-Stand automatisch

🌐 Was wird über das Netz übertragen?

DatenTechnikÜber Server?
🎵 Noten (welche Taste, wann, wie laut) WebSocket ✅ Ja — winzige Events (Bytes)
🎹 Welches Instrument aktiv WebSocket ✅ Ja — bei jeder Layer-Änderung
♩ Metronom BPM / Start/Stop WebSocket ✅ Ja
💬 Chat-Nachrichten WebSocket ✅ Ja
🎙 Mikrofon / Audio WebRTC (Peer-to-Peer) ❌ Nein — direkt zwischen Geräten

🔧 Layer-Einstellungen für Remote-Noten

Jeder Layer hat Quellen-Filter-Buttons (in den Layer-Details aufklappen):

🖥 Screen — eigene Klaviatur
⌨ Keys — PC-Tastatur
🎹 MIDI — USB-Gerät
▶ Player — Recorder
🌐 Remote — Jam-Noten

Wenn ein Layer auf 🌐 Remote reagieren soll, muss der Button aktiv (leuchtet) sein. Standardmäßig sind alle Quellen aktiv.

📶 Server-Optionen im Überblick

OptionVorteilNachteil
🖥 Laptop (Windows/Linux) Einfach, stabil, kein Aufwand Muss laufen wenn gejammt wird
📱 Android (Termux) Immer dabei, WLAN-unabhängig Akku-Optimierung ausschalten nötig
🥧 Raspberry Pi (später) 24/7, ~3W Stromverbrauch, feste URL möglich Einmalige Einrichtung nötig

localhost.run — Tunnel-Befehl

ssh -R 80:localhost:8765 nokey@localhost.run
# → wss://XXXXX.lhr.life  (in SF2 Player eintragen)
Tipp Raspberry Pi: Autostart einrichten:

sudo nano /etc/systemd/system/jam.service
→ Server startet automatisch beim Boot, localhost.run Tunnel als zweiter Dienst. Dann ändert sich die URL nie mehr.

❓ Häufige Probleme

ProblemLösung
Verbindung schlägt fehl URL prüfen: ws:// für lokal, wss:// für localhost.run
Noten hört man nicht SF2-Datei laden · Layer hat 🌐 Remote aktiviert · Mixer-Lautstärke > 0
Instrument klingt anders Preset-Chip erscheint grau (≈) → gleichen SF2 laden wie Mitspieler
Mikrofon-Erlaubnis verweigert Browser-Einstellungen → Kamera & Mikrofon → erlauben
Gitarren-Sustain wird abgehackt Modus auf 🎸 Instrument/Line-In stellen
Server stirbt auf Android Akku-Optimierung für Termux deaktivieren (Einstellungen → Apps → Termux)
localhost.run URL ändert sich Normal — URL nach jedem Neustart kurz teilen. Raspberry Pi = feste URL
